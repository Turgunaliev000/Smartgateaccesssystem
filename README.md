
# Smart Gate Access System

Рабочий проект системы цифрового доступа для КПП университета: Django backend + React/Vite frontend.

## Возможности

- Вход пользователя по email.
- Восстановление пароля по одноразовому коду из email.
- Открытие шлагбаума с записью события в журнал.
- Генерация гостевых QR-пропусков.
- Проверка гостевых QR-пропусков на КПП через страницу сканера.
- Сканирование QR через камеру браузера с ручным вводом как запасным вариантом.
- История въездов и действий.
- Уведомления с отметкой прочтения и удалением.
- Админ-панель для добавления и блокировки пользователей.
- SQLite база данных и Django admin.

## Запуск полного проекта через Django

```bash
python3 manage.py migrate
python3 manage.py seed_demo
npm run build
python3 manage.py runserver
```

Проект будет доступен на `http://127.0.0.1:8000`.

Django отдаёт интерфейс, API и SQLite базу с одного сервера:

- приложение: `http://127.0.0.1:8000`
- сканер QR: `http://127.0.0.1:8000/scanner`
- API: `http://127.0.0.1:8000/api/`
- Django admin: `http://127.0.0.1:8000/admin/`

## Запуск frontend отдельно для разработки

В отдельном терминале:

```bash
npm install
npm run dev
```

Frontend будет доступен на `http://localhost:5173`.

## Сканер камеры

На странице `/scanner` можно нажать `Сканировать камерой`. QR распознаётся через ZXing в Safari, Chrome и Яндекс Браузере. Браузер должен разрешить доступ к камере, а приложение должно быть открыто по HTTPS или на `localhost`. Если камера недоступна, код пропуска можно вставить вручную.

## Установка на телефон

Smart Gate работает как PWA:

- Android/Chrome: нажмите значок установки в шапке или выберите `Установить приложение` в меню браузера.
- iPhone/Safari: нажмите `Поделиться`, затем `На экран Домой`.

После установки приложение открывается полноэкранно с собственной иконкой. Для работы с актуальными данными и базой требуется соединение с Django-сервером.

## Развёртывание на Koyeb

Проект содержит `Dockerfile`, который собирает React и запускает Django через Gunicorn.

Переменные окружения:

- `DATABASE_URL` — строка подключения PostgreSQL.
- `DJANGO_SECRET_KEY` — секретный ключ Django.
- `DJANGO_DEBUG=false` — production-режим.
- `DJANGO_ALLOWED_HOSTS={{ KOYEB_PUBLIC_DOMAIN }}` — домен Koyeb.
- `SEED_DEMO=true` — необязательно; создаёт демонстрационные записи при запуске.

Для отправки кодов восстановления добавьте в Render или Koyeb:

- `EMAIL_HOST=smtp.gmail.com`
- `EMAIL_PORT=587`
- `EMAIL_HOST_USER` — адрес почтового ящика отправителя.
- `EMAIL_HOST_PASSWORD` — пароль приложения почтового ящика, не обычный пароль аккаунта.
- `EMAIL_USE_TLS=true`
- `DEFAULT_FROM_EMAIL` — тот же адрес отправителя.

Без SMTP-настроек локальный режим печатает письмо и код в терминал Django.

Контейнер автоматически применяет миграции перед запуском приложения.
